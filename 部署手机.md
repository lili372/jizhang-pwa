# 部署到手机 & 更新流程(安卓版)

自用场景。电脑做"服务器",安卓手机做"终端",数据全部留在手机浏览器里。

遇到问题或想了解原理看 `踩坑记录.md`。

---

## 一、首次部署

### 1. 电脑启动本地服务

```bash
cd "C:\Users\86132\Desktop\记账小工具"
python serve.py
```

命令行会停住不动,说明服务起来了,**这个窗口保持开着**。Windows 首次运行会弹防火墙提示,必须选"允许"。

### 2. 查电脑的局域网 IP

另开一个命令行:

```bash
ipconfig
```

找"无线局域网适配器 WLAN"下的 `IPv4 地址`,形如 `192.168.1.23`。

### 3. 手机连同一 WiFi,访问电脑

手机用 **Edge 或 Chrome**(不要用微信/QQ 内置浏览器),地址栏输入:

```
http://192.168.1.23:8000
```

(IP 换成你刚查到的)

### 4. 开启关键 flag

页面能打开,但还装不了 App,需要开一个浏览器开关:

1. 地址栏输入 `edge://flags`(Chrome 就是 `chrome://flags`)
2. 搜索框输入 `Insecure origins`
3. 找到 **"Insecure origins treated as secure"**,状态改成 `Enabled`
4. 下方文本框填入你的地址,**末尾不要带斜杠**:
   ```
   http://192.168.1.23:8000
   ```
5. 底部 **Relaunch** 重启浏览器,保险起见从任务栏彻底划掉再打开

### 5. 验证 Service Worker 注册成功

重新打开 `http://192.168.1.23:8000`,让页面完全加载完,然后:

- 地址栏输入 `edge://serviceworker-internals`(Chrome 就是 `chrome://serviceworker-internals`)
- 应该能看到 `192.168.1.23:8000` 的条目,状态 `ACTIVATED`

看到就继续。看不到参考 `踩坑记录.md` 的第 1、2 条排查。

### 6. 添加到桌面

- **Edge**:右上角 ⋯ → "**添加到手机**"
- **Chrome**:右上角 ⋮ → "**安装应用**"

(两个叫法不一样,别找错 → 看 `踩坑记录.md` 第 5 条)

桌面出现绿色"账"字图标,点开是独立 App 形态,**没有浏览器地址栏**。

### 7. 验证离线可用

关掉电脑的 `python serve.py`,或手机断开 WiFi,再点桌面图标——应该还能正常打开。

## 二、日常使用

- 平时直接点桌面图标,**不需要电脑、不需要 WiFi**
- 数据存在手机浏览器的 localStorage,不上传任何地方

---

## 三、数据安全两件事

**1. 路由器里锁定电脑 IP**(推荐)

路由器后台(通常 `192.168.1.1`)找"DHCP 静态 IP 绑定"/"地址保留",把电脑 MAC 地址和当前 IP 绑定。电脑每次连 WiFi 都会分到同一个 IP。

**2. 定期导出 JSON 备份**(兜底)

App → 设置 → 导出数据 → 得到 JSON → 发微信文件传输助手存档。建议:

- 每次更新代码前先导出
- 每隔一两个月至少一次

数据真丢了(比如 IP 变了),从备份 JSON 导入即可恢复。

> 为什么 IP 变了数据会"丢"?参考 `踩坑记录.md` 第 3 条。

## 四、后续代码更新流程

### 第 1 步 | 改 `sw.js` 里的版本号

```js
const CACHE = 'jizhang-v7';  // 比上次大 1
```

**必须做**。不改版本号手机永远用旧缓存。原理见 `踩坑记录.md` 第 4 条。

### 第 2 步 | 手机里导出一次数据

兜底。App → 设置 → 导出数据。

### 第 3 步 | 电脑起服务

```bash
cd "C:\Users\86132\Desktop\记账小工具"
python serve.py
```

端口默认 8000 不要改,IP 也要和手机 App 里的一致,否则 origin 不同数据就读不到。

### 第 4 步 | 手机连同一 WiFi,打开 App

App 会自动拉新文件,SW 发现版本号变了就会更新。

- 第一次打开可能还是旧界面 → **从任务栏把 App 划掉,再点图标**
- 实在不更新 → 浏览器设置里清该站点的**缓存**(不是 localStorage!)

### 第 5 步 | 验证

新功能正常 + 旧数据还在。

### 第 6 步 | 断 WiFi 再测

断网还能打开 + 数据还在,就结束了。电脑关掉服务。

## 六、常见问题速查

### 手机访问不到电脑 IP

- 电脑防火墙放行 Python 没?(首次运行会弹框,必须选"允许")
- 手机和电脑在不在同一个 WiFi?有些路由器"访客网络"会隔离设备
- IP 变了没?再 `ipconfig` 查一遍

### 菜单里找不到"安装应用 / 添加到手机"

参考 `踩坑记录.md` 第 2、5、6 条。快速检查:

1. flag 开了?`Insecure origins` → Enabled,URL 精确匹配
2. Relaunch 了?最好强制停止再打开
3. SW 注册了?`edge://serviceworker-internals` 必须有 `ACTIVATED` 条目
4. Edge 不行就试 Chrome

### 更新后还是旧界面

- `sw.js` 版本号改了吗?(最常见原因)
- 从任务栏彻底划掉 App 再重开
- 清浏览器里该站点的**缓存**(不是 localStorage!)
- 实在不行重装 App(**先导出数据备份!**)

### 换新手机

1. 旧手机:App → 设置 → 导出数据 → 得到 JSON
2. JSON 通过微信/网盘传到新手机
3. 新手机按"首次部署"流程装好 App → 设置 → 导入数据

### 电脑 IP 变了,手机 App 里数据没了

原理见 `踩坑记录.md` 第 3 条。处理:

- 把电脑 IP 改回去(路由器里绑静态 IP)
- 或直接用备份 JSON 导入

### 家里没 WiFi / 要出差

- 不用联网,App 本身就能用(SW 已缓存)
- 只在"更新代码"那几分钟需要电脑和手机在同一网络
- 也可以开手机热点让电脑连上,效果一样

## 七、未来给爸妈用的路径(暂存)

当前方案需要电脑开机、需要用户懂"同 WiFi 连 IP",不适合爸妈。真要给他们用时:

1. 代码推到 GitHub
2. 开 GitHub Pages / Cloudflare Pages / Vercel(任一免费静态托管)
3. 得到固定 https URL(比如 `https://xxx.pages.dev`)
4. 微信发链接给爸妈 → 他们点开 → 右上角"在浏览器打开" → 添加到桌面
5. 以后 `git push`,他们下次打开自动更新

https 下 PWA 100% 能装,不需要 flag,不需要电脑开机。
